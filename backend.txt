# Voice AI Sales Agent Backend - Frontend Integration Documentation

## 🎯 Overview

This document provides complete frontend integration details for the Voice AI Sales Agent backend. The system is a comprehensive multi-channel communication platform with AI-powered sales agents, real-time call management, and advanced content generation capabilities.

## 🏗️ System Architecture

### Core Components
- **Authentication System**: JWT-based with organization support
- **Organization Management**: Multi-tenant with role-based access
- **Agent Management**: AI-powered voice/email/SMS agents with Retell integration
- **Lead Management**: Contact management with assignment system
- **Call Management**: Voice calls via Retell AI with real-time AI responses
- **Content Generation**: AI-powered template and script generation
- **Communication Scheduling**: Time zone aware automation and campaigns
- **Analytics & Reporting**: Real-time usage and performance metrics

### Tech Stack
- **Backend**: FastAPI (Python)
- **Database**: Supabase (PostgreSQL)
- **Authentication**: JWT tokens
- **Voice**: Retell AI integration
- **AI**: OpenAI GPT integration
- **Real-time**: WebSocket support

## 🚀 Frontend Implementation Guidelines

### 1. Authentication Flow
```javascript
// Login flow
const login = async (email, password) => {
  const response = await fetch('/api/auth/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email, password })
  });
  const data = await response.json();
  
  // Store token
  localStorage.setItem('access_token', data.access_token);
  
  // If no org_id, user needs to create organization
  if (!data.org_id) {
    redirectTo('/organization/create');
  }
};
```

### 2. API Client Setup
```javascript
class ApiClient {
  constructor() {
    this.baseURL = 'http://localhost:8000';
    this.token = localStorage.getItem('access_token');
  }

  async request(endpoint, options = {}) {
    const url = `${this.baseURL}${endpoint}`;
    const config = {
      headers: {
        'Content-Type': 'application/json',
        ...(this.token && { 'Authorization': `Bearer ${this.token}` }),
        ...options.headers,
      },
      ...options,
    };

    const response = await fetch(url, config);
    
    if (response.status === 401) {
      // Token expired, redirect to login
      localStorage.removeItem('access_token');
      window.location.href = '/login';
      return;
    }

    return response.json();
  }

  // Agent methods
  async createAgent(agentData) {
    return this.request('/api/agents/', {
      method: 'POST',
      body: JSON.stringify(agentData),
    });
  }

  async getAgents() {
    return this.request('/api/agents/');
  }

  // Call methods
  async initiateCall(leadId, agentId) {
    return this.request('/api/calls/initiate', {
      method: 'POST',
      body: JSON.stringify({ lead_id: leadId, agent_id: agentId }),
    });
  }
}
```

### 3. Real-time Updates
```javascript
// WebSocket for real-time call updates
const setupCallWebSocket = (agentId) => {
  const ws = new WebSocket(`ws://localhost:8000/api/calls/llm/websocket/${agentId}`);
  
  ws.onmessage = (event) => {
    const data = JSON.parse(event.data);
    updateCallUI(data);
  };
  
  return ws;
};
```

### 4. Error Handling
```javascript
const handleApiError = (error) => {
  if (error.status === 401) {
    // Unauthorized - redirect to login
    window.location.href = '/login';
  } else if (error.status === 403) {
    // Forbidden - show access denied
    showToast('Access denied', 'error');
  } else if (error.status >= 500) {
    // Server error
    showToast('Server error. Please try again.', 'error');
  } else {
    // Other errors
    showToast(error.detail || 'An error occurred', 'error');
  }
};
```

## 🔒 Security Considerations

### Authentication
- JWT tokens expire in 24 hours
- Implement token refresh mechanism
- Store tokens securely (httpOnly cookies recommended)

### API Security
- All endpoints require authentication except public ones
- Role-based access control enforced
- Input validation on all endpoints

### Data Protection
- No sensitive data in localStorage
- Implement proper CORS headers
- Use HTTPS in production
